# 安装路径：仅核心框架（兼容旧名称）

这是旧入口名的兼容页。

“仅核心框架” 的真实 profile 是 `minimal`。

在当前产品口径里，它表示三件事：

- 从小工作内核开始
- 正常扩展路径仍然是 `skills/local/<skill-id>/SKILL.md`
- 不要把这个旧名称再理解成另一条 workflow lane 或技能目录模式

如果你现在要安装或更新，请回到唯一公开入口：

- [`one-click-install-release-copy.md`](./one-click-install-release-copy.md)

然后在同一个入口里选择：

- `仅核心框架 + 可自定义添加治理`

如果你只是想看 `minimal` 的命令边界，再继续看：

- [`minimal-path.md`](./minimal-path.md)

如果普通本地 skill 后续还是不够，而你又真的需要高级 manifest 驱动 custom workflow，再继续看：

- [`custom-workflow-onboarding.md`](./custom-workflow-onboarding.md)
