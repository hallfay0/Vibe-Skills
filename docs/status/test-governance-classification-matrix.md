# Test Governance Classification Matrix

Updated: 2026-07-13

## Positioning

The default regression set is capability-based, not directory-based. Keep the always-on lane limited to `install_lifecycle`, `runtime_entry_truth`, and `routing_mainline`. Everything else should either be rewritten into one of those seams, downgraded to an audit lane, or deleted when it only preserves historical cleanup proof.

## Core Capability Seams

| capability | seam summary |
| --- | --- |
| `install_lifecycle` | Managed install, check, update, and uninstall create or remove the right owned surfaces and leave user-owned files alone. |
| `runtime_entry_truth` | Canonical entry, runtime packet freezing, and bounded stage execution emit the right receipts, truth artifacts, and stop conditions. |
| `routing_mainline` | The router picks the right bounded owner, preserves safe fallback behavior, and projects selection data without inventing execution truth. |
