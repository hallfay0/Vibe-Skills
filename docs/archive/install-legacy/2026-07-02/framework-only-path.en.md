# Install Path: Framework Only (Compatibility Name)

This is a compatibility landing page for the older entry name.

The real profile behind “Framework Only” is `minimal`.

In the current product story, that means:

- start from the small work kernel
- keep the normal extension path at `skills/local/<skill-id>/SKILL.md`
- do not treat this older name as a separate workflow lane or catalog mode

If you are installing or updating now, go back to the single public entry:

- [`one-click-install-release-copy.en.md`](./one-click-install-release-copy.en.md)

Then choose:

- `Framework Only + Customizable Governance`

If you only want the `minimal` command boundaries, continue with:

- [`minimal-path.en.md`](./minimal-path.en.md)

If a normal local skill later turns out not to be enough, and you truly need an advanced manifest-driven custom workflow, only then continue with:

- [`custom-workflow-onboarding.en.md`](./custom-workflow-onboarding.en.md)
